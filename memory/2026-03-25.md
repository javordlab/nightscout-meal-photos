## Operational Update: Timezone Handling
- **FIXED:** `photo_to_log_pipeline.js` bug that was incorrectly using UTC time for local log entries.
- **STRICT GUIDELINE:** Always use the host's system timezone for logging and calculations. Never hardcode specific geographical offsets (like `-07:00` or `-08:00`) unless explicitly required by a target API. Use dynamic offset calculation (e.g., `Intl.DateTimeFormat` or native `Date` methods that reflect the system's timezone).
