Full audit of all scheduled jobs. Key findings and changes:

### Dead Pipeline: Telegram getUpdates (ROOT CAUSE of missing photo URLs)
- **Root cause:** `telegram_ingest_updates.js` uses Telegram's `getUpdates` polling API, which is permanently dead because OpenClaw uses webhooks. Last raw update was March 28.
- **Consequence:** Photos sent by Maria arrive via webhook → HealthGuard logs food text ✅ but no photo URL ever gets written to health_log.md ❌.

## Photo Flow Architecture (Canonical — Updated 2026-04-02)

### How OpenClaw receives Telegram media
1. Maria sends a photo in the Food Log group (-5262020908).
2. Telegram delivers the message via **webhook** to OpenClaw (not polling).
3. OpenClaw automatically downloads the file to `/Users/javier/.openclaw/media/inbound/file_NNN---uuid.jpg` **before** the agent runs.
4. The HealthGuard agent (`agent:health-guard`) is invoked with the message context (photo already on disk).

### HealthGuard's responsibility (real-time, same turn)
1. Analyze photo with vision model → estimate nutrition.
2. Fetch current BG from Nightscout.
3. Write food entry to `health_log.md` including `[photo](URL)` — must upload first:
   ```bash
   curl -s -X POST "https://freeimage.host/api/1/upload" \
     -F "key=6d207e02198a847aa98d0a2a901485a5" \
     -F "source=@/Users/javier/.openclaw/media/inbound/file_NNN---uuid.jpg" \
     | grep -o '"url":"[^"]*' | cut -d'"' -f4
   ```
4. Readback verify the entry in health_log.md.
5. Run `node scripts/radial_dispatcher.js` → pushes to Nightscout + Notion.
6. Reply to Maria with BG + confirmation.

### What does NOT work (dead — do not resurrect)
- `telegram_ingest_updates.js` — polls `getUpdates`, which returns nothing when webhook is active. **Dead since late March 2026.**
- `telegram_classify_updates.js` — downstream of above; `telegram_media_envelopes.jsonl` has not received a new entry since 2026-03-28.
- `photo_to_log_pipeline.js` — depends on envelopes for caption/timestamp matching. Without envelopes, skips every photo permanently.
- `auto_fix_pending_photo_refs.js` — looks for `[📷](file_NNN)` temp refs in health_log.md that were never written in the first place.
- **All four scripts are disabled.** Do not re-enable or rebuild without replacing the getUpdates source entirely.

### Why getUpdates is permanently dead
Telegram only delivers each update once — either to `getUpdates` **or** to the webhook, never both. Since OpenClaw registers a webhook, all updates go there. Calling `getUpdates` returns an empty array indefinitely.
