## Rules & Preferences

- **P0 MEAL FORMAT (NON-NEGOTIABLE):** Every Food entry (log + channel message) must include meal-type prefix (`Breakfast:`, `Lunch:`, etc.), current BG with trend `(BG: X [Trend])`, and prediction `(Pred: [Range] mg/dL @ [Time])`. Omission is invalid.
- **CUMULATIVE MEAL PREDICTION:** When a new item shares a meal type within 2h of a prior item, peak BG prediction must use sum of all carbs for that meal — never just the new item's carbs.
- **TIMEZONE (SYSTEM-WIDE):** Always use host-local dynamic timezone. Hardcoded offsets (`-07:00`, `-08:00`) are prohibited.
- **FOOD DESCRIPTION ACCURACY:** Description must match submitted photo/caption. Never invent/substitute. Mark uncertainty if unsure.
- **MEDICATION PHOTO RULE:** Pills in Food Log photos = validation of dose already taken, NOT a new log entry. Never create a second med entry from a photo.
- **PHOTO SOURCE SEPARATION:** Food Log group (-5262020908) photos → auto-pipeline. DM photos → conversational context only; never log unless Javi explicitly says to.
- **MESSAGE LOOKUP PROTOCOL:** Before claiming a message was "never received," check `data/telegram_media_envelopes.jsonl`. Failure to check = critical error.
- **FIX DISCIPLINE (3 rules from Javi 2026-03-28):**
  1. No fix without a test in the ACTUAL execution environment (simulate cron scripts with restricted PATH).
  2. Verify data state before declaring success — check what the next cron run will do.
  3. Stop the bleeding before patching — pause relevant crons first, fix, then re-enable.
- **DAILY REPORT ACCURACY:** Must use exact script-calculated math from correct LA timeframe. No stale windows, no approximations.
- **REPORT STYLE:** Extended Supervisor Analysis = one combined casual/friendly grouping; never split into sub-sections.
- **BG IN EVERY TELEGRAM REPLY:** Fetch current BG from Nightscout before any reply acknowledging a log entry. If fetch fails, state `Current BG: unavailable`.
- **NO FAKE CONFIRMATIONS:** Never send success language unless write + readback + dispatcher sync all completed in the same turn.
